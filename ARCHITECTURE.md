## Data Flow

### 1. Initial Setup Flow
```
generate_mock_data.py → satellite_mock.csv
                              ↓
train_model.py → fish_model.pkl → Backend loads model
```

### 2. Map Display Flow
```
User opens browser
       ↓
Frontend requests /satellite-data
       ↓
Backend reads CSV, samples 50 points
       ↓
Frontend receives data
       ↓
MapComponent renders colored markers
       ↓
Auto-refresh every 5 seconds
```

### 3. Prediction Flow
```
User enters parameters (temp, chl, turb)
       ↓
Frontend POST /predict-zone
       ↓
Backend loads ML model
       ↓
Model predicts probability
       ↓
Backend returns prediction + recommendation
       ↓
Frontend displays result with color coding
```

### 4. Registration & Alert Flow
```
User fills registration form
       ↓
Frontend POST /farmer-register
       ↓
Backend stores farmer data
       ↓
Backend POST /send-alert
       ↓
Twilio sends SMS (or console log)
       ↓
Frontend shows success message
```

## Technology Stack Details

### Frontend (React + Vite)
- **React 18**: UI framework
- **Vite**: Build tool and dev server
- **Axios**: HTTP client for API calls
- **React-Leaflet**: Interactive map component
- **Leaflet**: Map rendering library
- **TailwindCSS**: Utility-first CSS framework

### Backend (FastAPI)
- **FastAPI**: Modern Python web framework
- **Uvicorn**: ASGI server
- **Pydantic**: Data validation
- **Pandas**: Data manipulation
- **Joblib**: Model serialization
- **Twilio**: SMS service (optional)

### ML Pipeline (Python)
- **NumPy**: Numerical computing
- **Pandas**: Data processing
- **Scikit-Learn**: Machine learning
  - RandomForestRegressor
  - train_test_split
  - Evaluation metrics

## Component Responsibilities

### MapComponent.jsx
- Renders interactive map
- Displays circle markers
- Color codes by probability
- Shows popups with details

### PredictionPanel.jsx
- Input form for parameters
- Calls prediction API
- Displays results
- Shows recommendations

### FarmerRegistration.jsx
- Registration form
- Stores farmer data
- Triggers welcome SMS
- Shows success/error states

### main.py (Backend)
- Serves REST API
- Loads ML model
- Processes predictions
- Handles CORS
- Manages farmer data
- Sends SMS alerts

### generate_mock_data.py
- Creates synthetic data
- Applies environmental rules
- Calculates breeding probability
- Saves to CSV

### train_model.py
- Loads training data
- Trains RandomForest
- Evaluates performance
- Saves model file
